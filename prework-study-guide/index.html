HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Meta tags contain meta-data about the website.
  • The title shows on the browser tab and also helps with SEO.
  • Heading tags range from h1 to h6, which displays headings from biggest to smallest respectively.
  • Img tags are self-closing, and they have an image source (src) and an alt text.
  • Alt texts are extremely important for user accessibility and can also help with SEO.
  • It is best practice to use semantic HTML tags as often as possible, like body, header, section, footer, etc.
  • IDs are unique identifiers for elements.
  • Classes are identifiers similar to IDs, but they are not unique. Multiple elements can have the same class. This helps with CSS styling.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The CSS box model: all HTML elements are arranged like a box. Interactable info in Chrome Devtools.
  • CSS rule: Selector {property: value; property: value;}
  • A section of property:value is called a declaration.
  • A section of {property: value; property: value;} is called a Declaration block.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull branch-name: syncs our local code to the remote branch
  • git add -A: commit all files in the working branch
  • git commit -m: commits your changes to the repo with a message describing the changes
  • git push: pushes all your changes to the repo
  • Git and Github can be an essential tool for debugging, as you can look back on your previous code.

JavaScript

  • JavaScript is an Object Oriented Programming (OOP) language.
  • A variable is a named container that allows us to store data in our code.
  • Variables can store these primitive data types: String, Number, BigInt, Boolean, Undefined, Null, Symbol, Object
  • They can store structural data types: Objects, Arrays, and Functions
  • They can also be other data types like date or an error object.
  • Variables can be called with var, let, or const. Each have different scopes and changeability.
  • Control flow is the order in which a computer executes code in a script.
  • Conditional Statements and Loops can change the control flow.
  • Conditional logic is written with if/else if/else chains, each dealing statement that will return a boolean value
  • The boolean value can either be a truthy or a falsey. Falsey values are generally: 0, -0, On, "", Null, undefined, NaN (Not a Number)
  • A falsey value can also be any value the developer deems as False.
  • The conditions go in paranthesis () and the block of code to run given the condition goes in curly brakcets {}
  • Arrays are like variables, but can store multiple values of different data types.
  • Array items are in square brakcets [] and are comma separated.
  • Arrays have an indexing system that starts from 0.
  • For loops have 3 parts: a starting iteration variable, a boolean statement that determines the end of the loop, and the increment or decrement of the iteration.
  • For loops can be used to loop through an array by setting the index as the iteration variable and running through the length of the array using the .length attribute/property.
  • Functions are helpful for running large chunks of code at a time.
  • They are usually declared with the keyword "function", with the name of the function following afterwards, and then paranthesis ()
  • The function parameters go in the paranthesis ()
  • To run the function, you have to call the function like so: nameOfFunction(functionParameter);
