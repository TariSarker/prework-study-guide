Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Meta tags contain meta-data about the website.
  • The title shows on the browser tab and also helps with SEO.
  • Heading tags range from h1 to h6, which displays headings from biggest to smallest respectively.
  • Img tags are self-closing, and they have an image source (src) and an alt text.
  • Alt texts are extremely important for user accessibility and can also help with SEO.
  • It is best practice to use semantic HTML tags as often as possible, like body, header, section, footer, etc.
  • IDs are unique identifiers for elements.
  • Classes are identifiers similar to IDs, but they are not unique. Multiple elements can have the same class. This helps with CSS styling.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The CSS box model: all HTML elements are arranged like a box. Interactable info in Chrome Devtools.
  • CSS rule: Selector {property: value; property: value;}
  • A section of property:value is called a declaration.
  • A section of {property: value; property: value;} is called a Declaration block.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull branch-name: syncs our local code to the remote branch
  • git add -A: commit all files in the working branch
  • git commit -m: commits your changes to the repo with a message describing the changes
  • git push: pushes all your changes to the repo
  • Git and Github can be an essential tool for debugging, as you can look back on your previous code.
